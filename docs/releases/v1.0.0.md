# codex-profile-manager v1.0.0

The first public release of an unofficial manager for multiple Codex CLI authentication profiles with shared local context.

## Highlights

- Manage any number of numeric or named Codex profiles.
- Keep `auth.json` isolated per profile.
- Share only an explicit allowlist of sessions, history, configuration, memories, and known state files.
- Select a profile directly or rotate through profiles in configured order.
- Resume the latest session or start a fresh session through `codexpm`.
- Prevent concurrent managed Codex processes from writing to the same shared state.
- Validate backups, free disk space, paths, permissions, links, and authentication isolation.
- Migrate legacy `codex2keys` installations without modifying the original Codex binary.
- Preserve compatibility commands: `codex_smart`, `codex_switch`, and `codex_add_account`.

## Main commands

```bash
codexpm list
codexpm status
codexpm use <profile>
codexpm next
codexpm add <profile>
codexpm login <profile>
codexpm logout <profile>
codexpm run
codexpm run new
codexpm doctor
```

## Safety model

Only the first profile may inherit the original Codex authentication during initial installation. Additional profiles start without `auth.json` and must be logged in independently.

Unknown future Codex files are not shared automatically.

## Compatibility

The project currently targets Linux with Bash 4+ and standard Unix utilities, including `flock`.
